## INTRODUCTION
Les licences jouent un rôle fondamental dans le monde du logiciel. Elles définissent les droits et obligations des utilisateurs, des développeurs et des distributeurs concernant l'utilisation, la modification et la redistribution des programmes. Qu'il s'agisse de logiciels propriétaires ou open source, chaque licence représente un cadre légal qui détermine comment un logiciel peut être utilisé et partagé.  

**1. Protéger les droits d'auteur**  
Les licences assurent que le travail des développeurs est respecté et protégé par la loi.
**2. Encadrer l'utilisation**  
Elles définissent clairement ce que les utilisateurs peuvent ou ne peuvent pas faire avec un logiciel.
**3. Favoriser l'innovation**  
Certaines licences encouragent la collaboration et le partage de code, tandis que d'autres protègent des modèles économiques spécifiques.
**4. Réduire les risques juridiques**  
Un cadre clair permet d’éviter les litiges en cas de mésentente ou d’utilisation abusive.  

### Classification des licences
Les licences logicielles peuvent être regroupées en trois grandes catégories.  

- Licences propriétaires:  
Elles restreignent l'accès au code source et imposent souvent des limites strictes sur l'utilisation et la redistribution.
- Licenses open-source:  
Elles mettent l'accent sur la transparence et l'ouverture, mais n'imposent pas nécessairement de partager les modifications (sauf selon les termes spécifiques de la licence).
- Licenses libres:  
Elles concernent la liberté d'utiliser, modifier, partager et redistribuer le code, souvent avec des exigences d'ouverture (copyleft).  

Les licenses open-source et libre offrent des droits étendus, souvent avec l'objectif de promouvoir le partage et la collaboration.

## OPEN-SOURCE ET LIBRES
L'univers de l'open source repose sur le partage, la collaboration et la transparence. Cependant, ces principes ne peuvent fonctionner sans un cadre juridique clair pour protéger les droits des créateurs tout en offrant des libertés aux utilisateurs. C'est ici qu'interviennent les licences open source, des documents légaux qui définissent les conditions d'utilisation, de modification et de redistribution d'un logiciel.  

**1. Protéger les auteurs**  
Elles garantissent que le travail des développeurs est utilisé dans les conditions qu'ils définissent.
**2. Assurer les droits des utilisateurs**  
Elles précisent ce que les utilisateurs peuvent faire avec le logiciel (ex: utiliser, modifier ou distribuer).
**3. Éviter les litiges**  
Elles établissent un cadre légal qui réduit les risques de conflits.
**4. Encourager l'innovation**  
En facilitant le partage et la réutilisation des logiciels elles stimulent la collaboration et le développement technologique.  

Ces licences open-source/libre peuvent être regroupées en deux grandes catégories.  

- Licences permissives:  
Elles offrent une grande liberté, permettant d'utiliser, modifier et redistribuer le logiciel sans beaucoup de restrictions.  
Ex: MIT, BSD, ISC.  

- Licences avec copyleft:  
Le logiciel est bien soumis au droit d'auteur, mais plutôt que d'utiliser ces droits pour restreindre l'usage que peuvent en faire les utilisateurs ceux-ci sont utilisés pour assurer que tous les utilisateurs auront ces libertés.
Elles imposent donc que les travaux dérivés soient distribués sous la même licence, garantissant que le code reste libre.  
Ex: GPL, AGPL, LGPL.  
**Copyleft fort**  
Contraint tout projet dérivé à adopter la même licence.
**Copyleft faible**  
Permet un mélange avec du code propriétaire, sous certaines conditions.
**Copyleft modéré**  
S'applique uniquement aux fichiers modifiés ou spécifiques sous licence.

## HYBRIDE
### MIT
La licence MIT (Massachusetts Institute of Technology) est une des licences open source les plus simples et permissives. Elle est simple et concise, ne nécessitant que la mention de la licence dans les redistributions. La licence MIT est très populaire pour sa simplicité.  

| Caractéristiques | MIT |
| :---: | :---: |
| `Liberté d'utilisation` | Vous pouvez utiliser le logiciel pour n'importe quel usage, personnel ou commercial. |
| `Attribution` | Nécessite de conserver une copie de la licence d'origine et le copyright dans toute redistribution du code source. |
| `Modifications` | Vous êtes libre de modifier, copier et distribuer le code source, même en intégrant ces modifications dans un projet propriétaire. |
| `Redistribution` | Le code source redistribué doit être accompagné de la même licence MIT. |
| `Brevets` | L'auteur vous accorde une licence sur les brevets qu'il détient, couvrant le code source. |
| `Clause de "Notice"` | La licence et la notice doivent être incluses dans toute copie ou distribution du code source. |
| `Garanties` | Le logiciel est fourni "tel quel" sans aucune garantie. L'auteur décline toute responsabilité en cas de problème ou de dommages causés par l'utilisation du logiciel. |
| `Projets propriétaires` | Les modifications peuvent être intégrées dans des projets propriétaires sans obligation de rendre le code source public. |
| `Utilisation commerciale` | La licence ne donne pas le droit d'utiliser les marques déposées de l'auteur ou du logiciel sans autorisation. |
| `Propriété intellectuelle` | Possibilité de conserver la propriété du code source original si la mention est explicitement énoncée dans le fichier de license. |

## OPEN-SOURCE
### Apache 2.0
La licence Apache 2.0 est une licence open source largement utilisée et plus détaillée que la licence MIT. Elle offre également des droits permissifs tout en introduisant certaines clauses importantes notamment sur les contributions et les brevets.

| Caractéristiques | Apache 2.0 |
| :---: | :---: |
| `Liberté d'utilisation` | Vous pouvez utiliser le logiciel pour n'importe quel usage, personnel ou commercial. |
| `Attribution` | Exige d'inclure une notice de copyright, une copie de la licence et toute attribution liée à la contribution. |
| `Modifications` | Vous êtes libre de modifier, copier et distribuer le code source, y compris dans des projets dérivés. |
| `Redistribution` | Les copies redistribuées doivent inclure un avis de licence et les modifications doivent être documentées. |
| `Brevets` | L'auteur vous accorde une licence explicite sur les brevets couvrant le code source vous permettant de l'utiliser sans crainte de violations. |
| `Clause de "Notice"` | Le code source redistribué doit inclure une copie de la licence Apache 2.0 et un avis de modification si des changements ont été effectués. |
| `Garanties` | Le logiciel est fourni "tel quel", sans aucune garantie. L'auteur décline toute responsabilité en cas de problème ou de dommages causés par l'utilisation du logiciel. |
| `Projets propriétaires` | Les projets dérivés peuvent être propriétaires, mais doivent respecter les termes de la licence Apache 2.0, notamment la fourniture de la licence et la gestion des brevets. |
| `Utilisation commerciale` | La licence ne permet pas l'utilisation des marques déposées de l'auteur sans autorisation préalable. |
| `Propriété intellectuelle` | L'auteur conserve les droits d'auteur et les brevets liés au code source original. |

### BSD
La licence BSD (Berkeley Software Distribution) est une licence permissive qui permet une grande liberté d'utilisation, de modification et de redistribution des logiciels. Elle est souvent utilisée pour des projets open source, en particulier dans les systèmes Unix-like.  
Contrairement à d'autres licences comme la GPL, la licence BSD ne impose pas de conditions strictes sur la redistribution des modifications, ce qui permet aux développeurs de les intégrer dans des logiciels propriétaires.  

| Caractéristiques | BSD |
| :---: | :---: |
| `Liberté d'utilisation` | Vous pouvez utiliser le logiciel pour n'importe quel usage, personnel ou commercial, sans restrictions. |
| `Attribution` | Nécessite de conserver le copyright et une mention d'attribution dans la redistribution. |
| `Modifications` | Vous êtes libre de modifier le code source. |
| `Redistribution` | La redistribution doit inclure un avis de copyright, mais aucune obligation de redistribuer le code source modifié. |
| `Brevets` | Il n'y a pas de clause explicite concernant les brevets dans la licence BSD, mais les droits d'auteur restent protégés. |
| `Clause de "Notice"` | Toute redistribution du code doit inclure un avis de copyright et un avis de non-responsabilité. |
| `Garanties` | Le logiciel est fourni "tel quel", sans aucune garantie. L'auteur décline toute responsabilité en cas de problème ou de dommages causés par l'utilisation du logiciel. |
| `Projets propriétaires` | Les modifications peuvent être intégrées dans des projets propriétaires sans obligation de rendre le code source public. |
| `Utilisation commerciale` | La licence ne permet pas l'utilisation des marques déposées sans autorisation. |
| `Propriété intellectuelle` | L'auteur conserve les droits d'auteur, mais permet une utilisation et une redistribution libres du code. |

## LIBRE
### GPLv3
**Copyleft fort**
La licence GPLv3 (GNU General Public License version 3) est une licence de logiciel libre qui garantit aux utilisateurs la liberté d'étudier, de partager et de modifier un logiciel.  
Sa principale caractéristique est le copyleft, une condition stipulant que toute redistribution ou modification du code source doit être faite sous la même licence GPL, ce qui assure que le logiciel et ses dérivés restent libres.  
La GPLv3 inclut également des protections contre les violations de brevets, interdisant aux contributeurs d'engager des poursuites pour violation de brevet. Elle vise à renforcer la liberté des utilisateurs tout en prévenant l'appropriation privée des logiciels libres.

| Caractéristiques | GPLv3 |
| :---: | :---: |
| `Liberté d'utilisation` | Vous pouvez utiliser le logiciel pour n'importe quel usage, personnel ou commercial. |
| `Attribution` | Nécessite de fournir le texte complet de la licence et de préserver les mentions d'attribution d'origine. |
| `Modifications` | Vous êtes libre de modifier le code source, mais vous devez distribuer le code modifié sous la même licence GPLv3. |
| `Redistribution` | La redistribution du code source doit inclure la licence GPL et, si des modifications ont été faites, ces dernières doivent être clairement mentionnées. |
| `Brevets` | Vous êtes accordé une licence sur les brevets détenus par l'auteur du code source, couvrant le code, mais cette licence est révoquée si vous intentez une action en justice pour violation de brevet contre un contributeur du code. |
| `Clause de "Notice"` | Toute copie du code source, modifiée ou non, doit inclure un avis de licence GPL et indiquer les modifications apportées. |
| `Garanties` | Le logiciel est fourni "tel quel", sans aucune garantie. L'auteur décline toute responsabilité en cas de problème ou de dommages causés par l'utilisation du logiciel. |
| `Projets propriétaires` | Les projets dérivés doivent être distribués sous la même licence GPL, ce qui signifie que les modifications doivent également être ouvertes. Les projets propriétaires ne sont pas permis. |
| `Utilisation commerciale` | La licence ne permet pas l'utilisation des marques déposées de l'auteur sans autorisation préalable. |
| `Propriété intellectuelle` | L'auteur conserve les droits d'auteur sur le code source, mais accorde aux utilisateurs des droits d'utilisation, de modification et de redistribution sous les termes de la GPL. |

### LGPL
**Copyleft faible**
La license LGPL (Lesser General Public License) plus permissive de la GPL. Elle permet l'utilisation du code dans des logiciels propriétaires, à condition que les modifications apportées au code LGPL soient ouvertes. C'est une bonne option pour les bibliothèques logicielles qui peuvent être intégrées dans des projets commerciaux.  

| Caractéristiques | LGPL |
| :---: | :---: |
| `Liberté d'utilisation` | Modifications	Vous pouvez modifier le code source, mais vous devez rendre publiques les modifications que vous apportez au code LGPL. |
| `Attribution` | Exige d'inclure une copie de la licence et de maintenir les mentions d'attribution d'origine pour les parties modifiées ou redistribuées. |
| `Modifications` | Vous pouvez modifier le code source, mais vous devez rendre publiques les modifications que vous apportez au code LGPL. |
| `Redistribution` | Le code source redistribué doit être accompagné de la même licence LGPL. Si vous redistribuez un logiciel modifié, vous devez fournir le code source des modifications. |
| `Brevets` | L'auteur accorde une licence sur les brevets qu'il détient, couvrant le code source LGPL. Les contributeurs ne peuvent pas poursuivre en justice pour violation de brevet. |
| `Clause de "Notice"` | La licence et la mention de copyright doivent être incluses dans toute copie ou redistribution du code source. |
| `Garanties` | Le logiciel est fourni "tel quel", sans aucune garantie. L'auteur décline toute responsabilité en cas de dommage ou problème. |
| `Projets propriétaires` | Vous pouvez intégrer le code LGPL dans des projets propriétaires, mais les modifications au code LGPL doivent être publiées sous LGPL. Le code propriétaire n'est pas soumis à la licence. |
| `Utilisation commerciale` | L'utilisation des marques ou des logos de l'auteur est généralement interdite sans autorisation explicite. |
| `Propriété intellectuelle` | Les droits d'auteur sont maintenus par l'auteur ou le contributeur, mais la licence LGPL accorde des droits étendus pour l'utilisation, la modification et la redistribution. |

### MPL 2.0
**Copyleft modéré**
La MPL (Mozilla Public License) est une licence copyleft modérée, permettant de combiner le code MPL avec du code propriétaire dans un même projet. Elle exige que les fichiers modifiés sous MPL soient redistribués sous la même licence, mais permet de garder d'autres parties du code sous une licence différente.  

| Caractéristiques | MPL 2.0 |
| :---: | :---: |
| `Liberté d'utilisation` | Vous pouvez utiliser le logiciel pour n'importe quel usage, personnel ou commercial. |
| `Attribution` | Implique d'inclure une copie de la licence dans les fichiers modifiés sous MPL et de préserver les attributions d'origine. |
| `Modifications` | 	Vous êtes libre de modifier le code source, mais vous devez redistribuer toute modification sous la même licence MPL. |
| `Redistribution` | Si vous redistribuez le code, vous devez inclure le texte de la licence MPL, ainsi que le code source modifié, si applicable. |
| `Brevets` | L'auteur accorde une licence sur les brevets qu'il détient pour le code source. Les contributeurs ne peuvent pas poursuivre en justice pour violation de brevet. |
| `Clause de "Notice"` | Vous devez inclure une copie de la licence MPL et mentionner les modifications apportées dans toute redistribution du code source. |
| `Garanties` | Le logiciel est fourni "tel quel", sans garantie d'aucune sorte. L'auteur décline toute responsabilité en cas de dommage. |
| `Projets propriétaires` | Vous pouvez intégrer du code MPL dans des projets propriétaires, mais les fichiers modifiés doivent rester sous MPL. Le reste du code propriétaire peut rester sous une licence différente. |
| `Utilisation commerciale` | L'utilisation des marques ou des logos de Mozilla ou de l'auteur du logiciel est interdite sans autorisation explicite. |
| `Propriété intellectuelle` | La licence MPL permet de conserver les droits d'auteur tout en permettant la redistribution et la modification du code sous certaines conditions. |

### MPL
**Copyleft modéré**
La license MPL (Mozilla Open Source License) de redistribuer les modifications sous la même licence, mais les autres parties du code peuvent être sous une licence différente. Elle est souvent utilisée pour les projets de la Mozilla Foundation, comme Firefox.  

| Caractéristiques | MPL |
| :---: | :---: |
| `Liberté d'utilisation` | Vous êtes libre d'utiliser le logiciel à des fins personnelles ou commerciales. |
| `Attribution` | Idem que MPL 2.0, avec des exigences similaires en matière d'attribution dans les fichiers modifiés. |
| `Modifications` | Si vous redistribuez le code, vous devez inclure la licence MPL et la notice de copyright. Les fichiers modifiés doivent être sous MPL. |
| `Redistribution` | Si vous redistribuez le code, vous devez inclure la licence MPL et la notice de copyright. Les fichiers modifiés doivent être sous MPL. |
| `Brevets` | L'auteur accorde une licence sur les brevets qu'il détient pour le code source. Les contributeurs ne peuvent pas poursuivre pour violation de brevet. |
| `Clause de "Notice"` | Vous devez inclure la licence MPL et mentionner les modifications apportées dans toute redistribution du code source. |
| `Garanties` | Le logiciel est fourni "tel quel", sans garantie d'aucune sorte. L'auteur décline toute responsabilité en cas de dommages. |
| `Projets propriétaires` | Vous pouvez utiliser le code MPL dans un projet propriétaire, tant que les fichiers MPL modifiés sont sous MPL. D'autres parties du code peuvent être sous une licence différente. |
| `Utilisation commerciale` | L'utilisation des marques ou des logos associés à Mozilla ou à l'auteur du logiciel est interdite sans autorisation explicite. |
| `Propriété intellectuelle` | La licence MPL permet de conserver les droits d'auteur tout en permettant la redistribution et la modification du code sous certaines conditions. |

### Public Domain
Cette license (CC0, Unlicense) permet de renoncer complètement à tous les droits d'auteur et autres droits associés au code. Le logiciel peut être utilisé, modifié et redistribué sans aucune restriction. Des licences comme CC0 ou Unlicense sont utilisées par ceux qui souhaitent céder le code en tout ou partie au domaine public.  

| Caractéristiques | Public Domain |
| :---: | :---: |
| `Liberté d'utilisation` | Le logiciel peut être utilisé librement, sans aucune restriction, à des fins personnelles ou commerciales. |
| `Attribution` | Pas d'exigence d'attribution, sauf si explicitement mentionné par l'auteur. |
| `Modifications` | Le code peut être modifié, transformé et utilisé de manière illimitée. |
| `Redistribution` | Le logiciel peut être redistribué, même sans mention de l'auteur original, sous n'importe quelle forme. |
| `Brevets` | Aucun brevet n'est requis ou accordé, car le code est placé dans le domaine public. |
| `Clause de "Notice"` | Aucune obligation d'inclure une notice de copyright ou de licence. |
| `Garanties` | Le logiciel est fourni "tel quel", sans aucune garantie d'aucune sorte. L'auteur décline toute responsabilité en cas de dommages. |
| `Projets propriétaires` | Le code peut être intégré dans des projets propriétaires sans restriction. |
| `Utilisation commerciale` | L'utilisation des marques, logos ou noms associés au code est libre, mais cela ne signifie pas que l'auteur ait approuvé un tel usage. |
| `Propriété intellectuelle` | Le code est renoncé à tous les droits de propriété intellectuelle, et il est placé dans le domaine public. |

### CC0
La licence CC0 (Creative Commons Zero) permet à l'auteur de renoncer à tous les droits d'auteur et autres droits associés au code, afin que celui-ci puisse être utilisé par n'importe qui, pour n'importe quel usage, sans aucune restriction. Il s'agit en fait d'une licence qui place une oeuvre dans le domaine public.  

| Caractéristiques | CC0 |
| :---: | :---: |
| `Liberté d'utilisation` | Le logiciel peut être utilisé pour n'importe quelle fin, qu'elle soit personnelle ou commerciale, sans aucune restriction. |
| `Attribution` | Pas d'obligation d'attribution, mais une mention d'origine est encouragée par courtoisie. |
| `Modifications` | Le code peut être modifié, adapté, ou transformé sans aucune condition. |
| `Redistribution` | Le code peut être redistribué sous n'importe quelle forme, sans aucune obligation d'inclure des mentions ou conditions. |
| `Brevets` | Aucune gestion des brevets, car les droits de propriété intellectuelle sont renoncés. L'utilisateur n'a aucune restriction liée aux brevets. |
| `Clause de "Notice"` | Aucune obligation d'inclure une notice de copyright ou de licence dans les redistributions du code. |
| `Garanties` | Garanties	Le logiciel est fourni "tel quel", sans aucune garantie d'aucune sorte. L'auteur décline toute responsabilité pour les dommages. |
| `Projets propriétaires` | Le code peut être utilisé dans des projets propriétaires sans restriction ni obligation de publier les modifications. |
| `Utilisation commerciale` | Aucun droit sur les marques ou logos associés au code n'est accordé. Toute utilisation de marques nécessite une permission séparée. |
| `Propriété intellectuelle` | Le code est renoncé à tous les droits de propriété intellectuelle. L'auteur renonce à ses droits d'auteur et à tous les droits associés. |

### Unlicense
Similaire à CC0, la licence Unlicense permet également de renoncer à tous les droits d'auteur. C'est une licence qui place l'oeuvre dans le domaine public, sans restriction. Elle est particulièrement simple et explicite : il n'y a aucune condition.  

| Caractéristiques | Unlicense |
| :---: | :---: |
| `Liberté d'utilisation` | Le logiciel peut être utilisé pour n'importe quel usage, qu'il soit personnel ou commercial, sans aucune restriction. |
| `Attribution` | Pas d'exigence d'attribution, le code est librement utilisable sans mention d'origine. |
| `Modifications` | Le code peut être modifié, transformé ou adapté sans aucune condition. |
| `Redistribution` | Le code peut être redistribué sous n'importe quelle forme, sans obligation d'inclure une notice de copyright ou de licence. |
| `Brevets` | Aucun brevet n'est requis ou accordé, car tous les droits associés au code sont renoncés. |
| `Clause de "Notice"` | Aucune obligation d'inclure une notice de copyright ou une mention de licence dans les redistributions du code. |
| `Garanties` | Le logiciel est fourni "tel quel", sans aucune garantie d'aucune sorte. L'auteur décline toute responsabilité en cas de dommage. |
| `Projets propriétaires` | Le code peut être utilisé dans des projets propriétaires sans aucune restriction ni obligation de rendre le code source public. |
| `Utilisation commerciale` | L'utilisation des marques ou logos associés au code est libre, mais cela ne signifie pas que l'auteur a approuvé l'utilisation. |
| `Propriété intellectuelle` | Le code est renoncé à tous les droits de propriété intellectuelle et est placé dans le domaine public. |

### Beerware License
La Beerware License est très permisssive. Elle permet l'utilisation, la modification et la redistribution du code tant qu'une mention est faite dans la documentation ou le code source. Il est aussi stipulé qu'une "bière" doit être offerte à l'auteur lors d'une rencontre, une touche humoristique qui fait de cette licence un choix peu conventionnel.  

| Caractéristiques | Beerware License |
| :---: | :---: |
| `Liberté d'utilisation` | Le logiciel peut être utilisé pour n'importe quel usage, personnel ou commercial, sans restrictions. |
| `Attribution` | Simple mention du développeur dans les cas où la licence est respectée, avec la possibilité d'offrir une bière comme geste symbolique. |
| `Modifications` | Le code peut être modifié, transformé ou adapté sans condition, tant que l'auteur est crédité. |
| `Redistribution` | Le code peut être redistribué sous n'importe quelle forme, à condition de mentionner l'auteur dans la documentation. |
| `Brevets` | Aucun brevet n'est requis ou accordé par la licence, mais l'utilisateur ne peut pas violer de brevets existants. |
| `Clause de "Notice"` | Une mention de l'auteur doit être incluse lors de la redistribution ou de l'utilisation du code. |
| `Garanties` | Le logiciel est fourni "tel quel", sans aucune garantie d'aucune sorte. L'auteur décline toute responsabilité. |
| `Projets propriétaires` | Le code peut être utilisé dans des projets propriétaires, sans obligation de rendre le code source public. |
| `Utilisation commerciale` | L'utilisation des marques ou logos associés à l'auteur ou au projet n'est pas spécifiquement encadrée par la licence. |
| `Propriété intellectuelle` | L'auteur conserve les droits d'auteur, mais permet une utilisation libre et des modifications sous la condition de mentionner l'auteur. |
